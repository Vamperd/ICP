# 点云配准算法说明

本文档用于详细说明本仓库中 3 种自实现算法与 2 个基准脚本的作用、核心思路、关键步骤和结果含义，便于后续撰写实验报告时直接引用。

---

## 1. 实验任务与数据特点

本实验的数据位于 [ICP_data](c:/Users/86136/Desktop/大三春夏/大三春 智能移动技术/IPC/ICP_data)，共 10 帧二维激光点云：

- `0.ply` 到 `9.ply`
- 每帧大约 180 个点
- 点云本质上是二维扫描，只是以三维点格式存储，`z` 基本为 0
- 各帧之间具有连续运动关系，适合采用“上一帧结果作为当前帧初值”的顺序配准方式

本仓库的实现统一采用如下主流程：

1. 读取第 0 帧作为初始地图
2. 依次读取第 1 到第 9 帧
3. 将当前帧配准到当前累计地图
4. 记录累计位姿
5. 将当前帧并入地图
6. 输出最终地图、轨迹图、迭代误差图和结果表

## 3. 原始 ICP 算法说明

### 3.1 对应文件

- 算法函数：[myicp.m](c:/Users/86136/Desktop/大三春夏/大三春 智能移动技术/IPC/myicp.m)
- 主程序：[myicp_mapping.m](c:/Users/86136/Desktop/大三春夏/大三春 智能移动技术/IPC/myicp_mapping.m)

### 3.2 核心思想

原始 ICP 是最经典的点到点配准方法，其核心思想是：

1. 对当前帧中的每个点，在目标地图中寻找最近邻点
2. 把这些最近邻看作对应点
3. 通过 SVD 求解使源点集尽量对齐到目标点集的刚体变换
4. 重复迭代，直到误差足够小或收敛

### 3.3 在本项目中的具体实现

`myicp.m` 中的实现步骤为：

1. 读取源点云 `curr_ply` 和目标点云 `base_ply`
2. 使用上一帧累计位姿 `tform_init` 作为粗配准初值
3. 用 `knnsearch` 为当前帧中每个点寻找目标地图中的最近邻
4. 对匹配点对使用 SVD 求解旋转矩阵 `R` 和平移向量 `t`
5. 用新的 `R,t` 更新当前帧
6. 计算点到点残差并形成 `mse_history`
7. 当均方误差小于阈值，或前后两次误差几乎不变时停止

### 3.4 数学形式

给定匹配点对：

- 目标点：`p_i`
- 当前点：`p'_i`

目标是求解：

`min sum || p_i - (R p'_i + t) ||^2`

其中：

- `R` 是旋转矩阵
- `t` 是平移向量

求解方法：

1. 分别计算两组点的质心
2. 去中心化
3. 计算协方差矩阵 `H`
4. 对 `H` 做 SVD 分解
5. 由 `R = V * U'` 得到旋转
6. 由质心关系求平移

### 3.5 优点与不足

优点：

- 实现简单
- 可解释性强
- 与教材中的标准 ICP 一致

不足：

- 只使用点到点距离，容易在墙面、长边界上出现切向滑移
- 对错误最近邻较敏感
- 在二维激光场景下通常不如点到线方法稳定

## 4. PL-ICP 算法说明

### 4.1 对应文件

- 算法函数：[myplicp.m](c:/Users/86136/Desktop/大三春夏/大三春 智能移动技术/IPC/myplicp.m)
- 主程序：[myplicp_mapping.m](c:/Users/86136/Desktop/大三春夏/大三春 智能移动技术/IPC/myplicp_mapping.m)
- 参考实现：[SYK_ICP/src/pl_icp_kd.m](c:/Users/86136/Desktop/大三春夏/大三春 智能移动技术/IPC/SYK_ICP/src/pl_icp_kd.m)

### 4.2 为什么点到线更适合二维激光

二维激光扫描中的大量环境结构是墙面、边界和轮廓线。

如果使用点到点 ICP：

- 同一面墙上的不同点之间也可能被视作“不错的最近邻”
- 算法容易沿着墙面方向滑动

如果使用点到线 ICP：

- 当前点不再强行对齐到某一个离散点
- 而是对齐到目标扫描局部线段上的投影点
- 这样更符合真实几何关系

因此 PL-ICP 往往会让最终地图更密集、边界更整齐。

### 4.3 在本项目中的实现方式

`myplicp.m` 参考 `SYK_ICP` 的结构实现，主要步骤如下：

1. 对地图点云估计法向
2. 对当前帧点云估计法向
3. 对当前帧中每个点，在地图中寻找最近的两个点
4. 这两个最近点构成一条局部线段
5. 计算当前点到该线段的距离
6. 检查目标线段两端点法向与当前点法向是否一致
7. 若满足距离阈值与法向阈值，则将当前点投影到该局部线段上
8. 用这些投影点和源点做 SVD，更新刚体变换

### 4.4 法向估计

法向估计采用局部协方差矩阵方法：

1. 对每个点寻找局部邻域点
2. 计算邻域点的协方差矩阵
3. 取最小特征值对应的特征向量作为法向

在当前实现中，法向估计采用较大邻域：

- 先找 80 个近邻
- 再通过半径约束筛掉过远点

这样做的目的是：

- 获得更平滑的法向
- 减少法向抖动带来的错误筛选

### 4.5 目标函数含义

PL-ICP 压低的是“点到线距离”，不是纯粹的点到点距离。

这意味着：

- 它更重视法向方向上的几何一致性
- 但如果设计不当，也可能沿线方向滑移

因此在实验中必须结合：

- 最终地图是否更密集
- 轨迹是否合理
- 墙面是否更对齐

而不是只看 `MSE` 数值。

### 4.6 当前实现中的修正点

为了避免“数值下降但地图更差”的问题，当前 `myplicp.m` 做了这些修正：

- 法向估计改回大邻域方式
- 点线匹配逻辑尽量贴近 `SYK_ICP`
- 去掉过早停止，避免停在不好的局部最优

## 5. N-ICP 算法说明

### 5.1 对应文件

- 算法函数：[mynicp.m](c:/Users/86136/Desktop/大三春夏/大三春 智能移动技术/IPC/mynicp.m)
- 主程序：[mynicp_mapping.m](c:/Users/86136/Desktop/大三春夏/大三春 智能移动技术/IPC/mynicp_mapping.m)

### 5.2 设计动机

N-ICP 可以理解为“法向约束增强版 ICP”。

它的目标不是只看最近邻距离，而是同时利用：

- 空间距离
- 法向一致性
- 局部线结构
- 切向位移约束

这样可以进一步减少错误匹配。

### 5.3 本项目中采用的具体形式

当前 `mynicp.m` 实现的是一种“法向引导的点到线 ICP”，主要步骤如下：

1. 对地图点和当前帧点都估计法向
2. 对每个当前点，在地图中寻找多个候选锚点
3. 每个候选锚点再与其地图邻点构造一条局部线段
4. 对每条候选线段计算：
   - 点到线距离
   - 法向一致性
   - 切向位移
5. 根据综合评分选出最佳对应
6. 把最佳投影点作为目标点，与当前源点做 SVD
7. 更新刚体变换并继续迭代

### 5.4 为什么它比简单法向过滤更强

如果只做“最近邻点到点 + 法向过滤”，会有两个问题：

1. 一开始找错最近邻，就容易一直错下去
2. 只用单个目标点，约束仍然偏弱

而当前实现引入了：

- 多候选目标点
- 局部线段
- 切向约束

所以比简单的法向过滤版本更稳定。

### 5.5 评分策略

当前实现中，候选对应的评分主要由以下部分构成：

- 点到线距离
- 切向位移惩罚
- 法向不一致惩罚

最终选择评分最小的那组对应。

这意味着 N-ICP 同时兼顾了：

- 几何贴合
- 法向方向合理
- 不在墙面上乱滑

### 5.6 当前调参结论

经过小范围参数扫描，当前在 [mynicp_mapping.m](c:/Users/86136/Desktop/大三春夏/大三春 智能移动技术/IPC/mynicp_mapping.m) 中采用的较稳参数为：

- `dist_threshold = 1.0`
- `cos_threshold = 0.08`
- `tangent_threshold = 1.2`
- `candidate_k = 6`

这组参数是在保证轨迹长度合理的同时筛选出来的折中方案。

## 6. 三种算法的对比总结

### 6.1 原始 ICP

特点：

- 最简单
- 点到点
- 易于实现和理解
- 对错误最近邻敏感

适用：

- 作为基线方法
- 适合作业中展示标准 ICP 原理

### 6.2 PL-ICP

特点：

- 点到线
- 更适合二维激光
- 对墙面、边界更友好
- 通常地图更密集、更贴边

适用：

- 作为本实验的主要改进方法
- 适合展示“为什么二维激光应该优先考虑点到线”

### 6.3 N-ICP

特点：

- 在点到线基础上进一步加入法向约束
- 对错误匹配抑制更强
- 对参数更敏感
- 实现复杂度高于前两者

适用：

- 作为增强算法
- 适合在报告中展示进一步改进思路

## 7. 结果表中各指标的含义

每个自实现主程序都会打印 `metrics_table`，其包含以下 3 项：

### 7.1 `iteration_count`

表示这一对配准实际执行了多少轮迭代。

含义：

- 值较大说明这一对较难收敛
- 值较小说明初值较好或误差下降较快

### 7.2 `valid_correspondence_count`

表示该轮最终参与优化的有效匹配点数。

含义：

- 值越大说明保留下来的匹配越多
- 若过小，说明对应点筛选过严或配准质量较差

### 7.3 `mean_residual`

当前代码中它实际是最终匹配误差的 `RMSE`，即：

`sqrt(MSE)`

注意：

- 它不是简单平均欧氏距离
- 它也不一定能跨算法直接比较
- 尤其 PL-ICP 和 N-ICP 的残差更接近“点到线”意义，不应机械地和原始 ICP 的点到点残差做绝对比较

## 9. 适合写入实验报告的结论方向

在实验报告中可以重点围绕以下几个角度组织分析：

1. 原始 ICP 实现简单，但在二维激光场景中容易发生切向滑移
2. PL-ICP 利用局部线段约束，通常比点到点 ICP 更适合墙面结构
3. N-ICP 在 PL-ICP 基础上进一步引入法向约束，理论上更强，但参数更敏感
4. 对于二维激光点云，不能只看数值误差，还必须结合地图密集度和轨迹合理性综合判断
5. 当前实验说明：更复杂的方法不一定天然更好，必须结合场景和参数调优
